# 计算机联锁模拟仿真系统 — 功能与演示说明

> 技术栈：Vue 3 + Pinia + Naive UI + Vite。启动：`npm run dev` → http://localhost:5173

---

## 一、站场概况

典型单线三股道中间站，对照真实站场图（`img.png`）绘制：

```
            3G（上股）
JXG —— IIAG ——— IIG（正线/中股）——— IIBG —— JSG
            1G（下股）
                  └ 安全线(PZA)
```

| 设备 | 清单 |
|------|------|
| 轨道区段（8） | JXG、IIAG、3G、IIG、1G、SFXG(安全线)、IIBG、JSG |
| 道岔（5） | 1#、3#（进站咽喉）、2#、4#（出站咽喉）、5#（安全线） |
| 信号机（10） | X、S（进站）；X3、XII、X1、S3、S1（出站）；D1、D2、SII（调车） |
| 进路模板（13） | 接车/发车各方向 + 调车 |

---

## 二、已实现功能

### 1. 站场图（SVG，可交互）
- 棕黄轨道线、绝缘节（青色方块）、轨道电路（青色圆）、坡度 6‰、安全挡车器，按真实图绘制
- **道岔分支随位置接通/断开**：扳道岔后，接通的股道分支亮棕黄、断开的变灰
- **设备点击**：点道岔/信号机弹出操作浮层；点区段切换占用（受联锁约束）
- 状态配色：空闲(棕黄)、锁闭(橙)、占用(红)；信号灯 绿/黄/红/灰(故障)

### 2. 进路办理（进路面板）
- 13 条进路一键选排锁闭、开放信号、正常解锁、取消进路（30s 倒计时 + 人工解锁）
- 锁闭时自动转换并锁闭相关道岔、锁闭沿途区段
- 列车作业办理（接车/发车/通过）、调车作业办理（起止区段）

### 3. 道岔控制（道岔面板 / 点击浮层）
- 单操定位/反位、单锁、封锁、故障仿真
- 四重锁定：进路锁、单锁、封锁、故障，状态在图上与面板同步显示

### 4. 信号机（信号面板 / 点击浮层）
- **进站/出站信号由进路驱动**：不能手点，只能"办进路→开放信号"自动变绿，按钮置灰提示
- 调车信号可手动控制黄/红
- 断丝故障仿真（强制红灯）

### 5. 列车管理（列车面板）
- 添加列车（车次、初始区段、上/下行）、出清
- **手动移动**：沿进路相邻区段逐段推进
- **自动运行**：沿已开放进路 path 一节一节顺序前进，遵守联锁，到终点停（间隔秒数可调）
- 列车作业记录（入站/运行/出站）

### 6. 操作日志与监控（日志面板）
- 全部操作（成功/拒绝）带时间戳与级别记录，可按级别筛选、导出
- 设备状态统计、系统全体复位

### 7. 全局反馈
- 所有操作（含站场图点击）统一弹出全局消息条（绿=成功 / 红=拒绝），与日志同步

## 三、联锁约束（核心规则）

本系统采用**进路驱动**模型，信号、进路、占用三者严格耦合：

| 规则 | 表现 |
|------|------|
| **信号进路驱动** | 进站/出站信号不能手点，只有 `开放信号` 能让其变绿；解锁/驶过自动复红 |
| **红灯/无进路 → 不能占用** | 没办开放进路时点站内区段被拒"无开放进路，不能占用"；区间 JXG/JSG 自由 |
| **占用 → 信号锁定** | 列车占压期间，进站/出站信号不可改，只有系统自动复红 |
| **驶过 → 自动复红** | 列车驶离信号机内方第一区段，防护信号自动复红 |
| **占用 → 不能选排/解锁** | 区段被占用时该路径不能选排；进路须全段出清才能解锁 |
| **进路锁 → 道岔禁动** | 进路锁闭的道岔不能单操/单锁 |
| **进路兑现自动解锁** | 列车到达进路终点（末段占用、前段出清）→ 进路自动解锁，释放道岔/区段锁 |
| **发车始端可占用** | 发车进路允许始端股道有待发列车占用 |

仿真功能：道岔故障/封锁、信号断丝、轨道占压/出清，均可手动触发并受联锁约束。

## 四、演示流程：一列车完整通过（接车进站 → 停站 → 发车出站）

### 准备
- 打开 http://localhost:5173（首次或改版后按 Ctrl+Shift+R 强制刷新）

### A. 接车进站（X → IIG 正线）
1. **进路面板** → 选「X→IIG（接车进正线）」→ 点 **锁闭进路**
   - JXG/IIAG/IIG 变橙(锁闭)，1#道岔自动转定位并锁闭，弹绿色消息
2. → 点 **开放信号**
   - **X 进站信号自动变绿**（此时去点 X 信号机会发现手点不动，提示"由进路自动控制"）
3. **列车面板** → 添加列车 `T1`，初始区段 **JXG**，上行 → JXG 变红
4. 让列车前进（二选一）：
   - 手动：T1 卡片"移动至" → IIAG → IIG
   - 自动：设间隔(如 2 秒) → **启动自动运行**，列车沿 JXG→IIAG→IIG 逐段前进
5. 观察：
   - 列车驶离 JXG 时 **X 信号自动复红**（日志："列车驶过，X 信号自动复红"）
   - 列车到达 IIG（终点）→ **接车进路自动兑现解锁**（日志："进路兑现…自动解锁"），1#道岔释放，IIG 只剩占用(红)

### B. 停站
- 列车停在 IIG 正线，接车作业完成。此时可去日志面板查看整条操作链。

### C. 发车出站（IIG → S 方向）
6. **进路面板** → 选「IIG→S（正线发车下行）」→ 点 **锁闭进路**
   - 发车进路允许始端 IIG 有车，正常锁闭；2#/4#道岔自动转定位锁闭，IIBG/JSG 变橙
7. → 点 **开放信号** → **XII 出站信号自动变绿**
8. 让列车前进（手动"移动至"或自动运行）：IIG → IIBG → JSG
   - 列车驶离 IIG 时 XII 自动复红
   - 列车到达 JSG（终点）→ **发车进路自动兑现解锁**
9. **列车面板** → 出清 `T1`

至此完成 **接车 → 停站 → 发车** 的完整闭环，全程信号由进路驱动、约束可见、日志可查。

### 其他可演示点
- **约束被拒**：不办进路直接点 IIG 区段 → 红色消息"无开放进路，不能占用"；区间 JXG 可自由变红
- **信号进路驱动**：直接点 X 信号机绿灯 → 灰按钮，提示"由进路自动控制"
- **敌对进路**：办了 X→IIG 后再办 X→3G（需同一道岔反位）→ 被拒
- **道岔联锁**：进路锁闭后操作相关道岔 → 被拒
- **故障仿真**：道岔/信号设故障 → 相关进路无法办理
- **取消进路**：办进路后点取消 → 30s 倒计时，可立即人工解锁
- **系统复位**：日志面板"系统全体复位"恢复初始状态
